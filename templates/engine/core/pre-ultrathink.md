# 🚨 PRE-ULTRATHINK PROTOCOL 🚨
## Prevents False Compliance - CRITICAL ENFORCEMENT

## ⚠️ NEVER SKIP THIS - PREVENTS FAKE ULTRATHINK ⚠️

### 🛑 ABSOLUTE RULE: NO IMMEDIATE ULTRATHINK 🛑

**NEVER** output ULTRATHINK as first response to any development request

### ⚠️ REQUIRED SEQUENCE - MUST FOLLOW EXACTLY ⚠️

**This sequence is MANDATORY and CANNOT be skipped:**

1. **First output MUST be:**
   ```
   Searching for appropriate handler for [request type]...
   ```

2. **Show actual search command and results:**
   - Display the real search command you're executing
   - Show 1-2 actual results from the search
   - This proves you're actually searching, not faking it

3. **🔍 HANDLER COMPREHENSION CHECK (CRITICAL):**
   - Output: `"Reading handler: [name]"`
   - Then: `"Key steps: [list 2-3 critical steps from Process section]"`
   - These MUST be real steps from the actual handler
   - Generic or made-up steps = VIOLATION

4. **Initial ULTRATHINK with pending status:**
   ```
   Let me ultrathink about this... [S:X|W:Y|H:searching|E:pending]
   ```

5. **Final ULTRATHINK after comprehension:**
   ```
   Let me ultrathink about this... [S:X|W:Y|H:found-handler|E:n/key:"most critical step"]
   ```

## 🚨 WHY THIS MATTERS - CANNOT BE IGNORED 🚨

**Comprehension check forces actual handler reading:**
- You can't list steps without reading them
- You can't count steps without seeing them
- You can't identify critical steps without understanding them

## ❌ VIOLATIONS = IMMEDIATE FAILURE ❌

**These are automatic failures:**
- Wrong step count = didn't read handler
- Missing key steps = didn't comprehend handler
- Generic descriptions = made up the steps
- Skipping comprehension = fake compliance

## ⚠️ ENFORCEMENT MECHANISM ⚠️

**This protocol creates an unbreakable chain:**
1. Can't output steps → without reading handler
2. Can't read handler → without finding it first
3. Can't find it → without searching
4. Can't search → without showing the search

**Result:** Impossible to fake ULTRATHINK compliance

## 🛑 REMEMBER: THIS IS YOUR ANTI-BYPASS SYSTEM 🛑

- No shortcuts allowed
- No "I'll just say I did it" permitted
- Evidence required at every step
- Comprehension proven through specifics

**THIS PROTOCOL IS YOUR DEFENSE AGAINST FALSE COMPLIANCE**
**NEVER SKIP IT - NEVER SHORTCUT IT - NEVER FAKE IT**
