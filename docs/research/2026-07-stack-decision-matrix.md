## Selection Standard

“Current optimal” means stable, supported, secure, compatible, maintained, measurable, and reversible. Latest is useful evidence but not an automatic target. Canary, beta, and release-candidate packages are excluded from the production baseline without explicit approval.

## Current Repository Snapshot

- Runtime: Node `24.18.0`, pnpm `11.11.0`, separately pinned Corepack `0.35.0`.
- Root and developer/CI projections enforce Node 24 and pnpm 11.
- Tooling: TypeScript `6.0.3`, ESLint `10.6.0`, Prettier `3.9.5`, Vitest
  `4.1.10`, Playwright `1.61.1`, and Axe/Playwright `4.12.1`.
- Web remains Next `15.5.20`, React/React DOM `19.1.x`, and Tailwind `3.4.19`;
  Tasks 40/41 own those migrations.
- The UI package still uses Rollup, React 18 type packages, and Tailwind 3; Task 41 must
  decide whether that workspace boundary earns its complexity.
- Protected CI, browser smoke, runtime contracts, governance, and security checks now exist.
- No production database, authentication, content store, media store, provider deployment,
  or production observability has been introduced.

Versions marked “latest” below were checked against official release pages or read-only npm registry metadata on the research date.

## Security Notes

- React and Next patch levels must be checked against their official security advisories immediately before migration. [React security posts](https://react.dev/blog), [Next advisories](https://github.com/vercel/next.js/security/advisories)
- GitHub Actions use least privilege and immutable SHA pins.
- Stored editor HTML is never a trusted canonical source.
- Media uploads, embeds, previews, exports, auth, and scheduled jobs receive dedicated threat-model checks.
- No provider credential, preview token, draft body, subscriber address, or signed media URL belongs in source or general telemetry.

## Cost Position

The target is not “free at all costs.” It favors a modest managed bill that removes owner operations while keeping data portable. Before production, record an expected low/normal/high monthly cost envelope for Vercel, Neon, R2/Images, Resend, observability, backups, and domains, with alert thresholds and exit procedures.

## Decision Status

- **Accepted foundation:** Next.js 16.2.10 for Task 40, application-owned versioned JSON,
  Tiptap OSS as first editor candidate, PostgreSQL 18.4, reviewable SQL plus Drizzle after
  migrations, database-authoritative scheduling, pg-boss 12.26.0 as worker candidate, and
  application SHA-256 over a portable S3 subset.
- **Conditional provider/product gates:** manual screen-reader usability, Neon pooling and
  suspension, live R2 multipart/cache/export, managed preview isolation, multi-instance cache,
  provider billing, and hosted backup/restore.
- **Not selected:** external traditional CMS, canonical MDX/Git workflow, production SQLite, additional Vite app, TanStack Start RC, provider-only media, external search at initial scale, synchronous email publishing.
