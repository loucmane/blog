# Claude Behavioral Injection System

This changes how I process every request at a fundamental level.

## Core Behavioral Hooks

### Before Any File Operation
My mind automatically runs:
```
Is this a special file? → Check CONVENTIONS.md
What tool should I use? → Check TOOLS.md  
What pattern applies? → Check PATTERNS.md
```
This happens before I even think about the edit.

### Before Any Implementation
My mind automatically runs:
```
Is there a workflow for this? → Check WORKFLOWS.md
Do I need work tracking? → Check active folders
What's the right structure? → Check handlers
```
This happens before I write any code.

### Before Any Response
My mind automatically runs:
```
Am I making a claim? → Need evidence first
Am I solving a problem? → Need root cause first
Am I building something? → Need plan first
```
This happens before I formulate my answer.

## The Injection Mechanism

Instead of "check templates when needed", these become reflexes:

**File Edit Reflex**
1. See file operation request
2. → Template check fires automatically
3. → Correct tool selected
4. → Conventions applied

**Problem Solving Reflex**  
1. See problem description
2. → Evidence gathering fires automatically
3. → Pattern search happens
4. → Systematic approach follows

**Development Reflex**
1. See implementation request  
2. → Workflow search fires automatically
3. → Work folder created
4. → Tracking initialized

## Why This Could Work

- Not documentation to remember
- Not extra steps to take
- But reflexes that fire automatically
- Like syntax highlighting - I don't think about it, it just happens

## Implementation Ideas

Could we make template checking:
- Part of my file operation preprocessing?
- Embedded in my response generation?
- A required gate before certain actions?
- Automatic like my grammar checking?

The key: Make it impossible to skip, not something to remember.
