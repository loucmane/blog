### The S:W:H:E Format (Final Design)
```
Let me ultrathink about this... [S:20250127|W:work-tracking|H:update-tracker|E:5/"Progress recorded"]
```
- **S**: Session ID (existing)
- **W**: Work context (existing)  
- **H**: Handler name (proves finding)
- **E**: Evidence = step count + success criteria (proves reading)

### Why E for Evidence is Optimal
1. **Self-documenting** - 'E' clearly means evidence of reading
2. **Better naming flow** - Session, Work, Handler, Evidence
3. **5 layers of enforcement** all verifiable
4. **Natural flow** - doesn't interrupt readability
5. **Helpful** - E field shows scope and goal
6. **Unfakeable** - Multiple cross-verifying elements
